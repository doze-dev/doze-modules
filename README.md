# doze-modules

Out-of-process engine **plugins** for [doze](https://github.com/doze-dev/doze).
Each module is a pure-Go plugin binary that speaks doze's engine gRPC protocol;
doze fetches one by engine type from the **signed registry**, verifies the
publisher's **ed25519 signature** (and its `sha256`), caches it under
`~/.doze/modules`, and runs it as a subprocess. This is the Terraform *provider*
model: doze core is a thin host, every engine is a separately-versioned module.

> **How distribution actually works.** doze core fetches from the signed registry
> at `https://doze.nerdmenot.in/registry` and *requires* a valid signature on
> every artifact. This repo's CI builds the plugins and uploads them to GitHub
> releases; those releases are a **publish input**, not something doze consumes
> directly. [doze-registry](https://github.com/doze-dev/doze-registry) CI signs
> them into the registry (triggered automatically on release). A raw
> `DOZE_MODULES_MIRROR` pointed at an *unsigned* `dzm` dist is rejected at
> runtime unless its `index.yaml` carries signatures — use it only as a
> dev/`file://` mirror when you have a namespace key.

## Layout

```
modules.yaml          # the manifest: each module's source path (in doze) + version
cmd/dzm/              # the build tool: cross-compiles plugins + assembles the release layout
.github/workflows/    # release CI (single runner — plugins are pure-Go cross-compiles)
```

A module is **not** its own repo — it's an entry in `modules.yaml`. Authoring or
upgrading one is a PR editing that file; CI builds the four supported triples
(`{aarch64,x86_64}-{apple-darwin,unknown-linux-gnu}`) and publishes cumulatively.

## Two version axes

A module has a **module version** (its plugin binary's semver, in `modules.yaml`)
— a different axis from the **engine version** a user declares in `doze.hcl`
(`postgres "app" { version = 18 }`). Users only ever write the engine version;
doze picks the module release automatically: the newest one compatible with its
plugin protocol and the declared engine majors, frozen in `doze.lock` and moved
only by `doze modules upgrade`.

Each release's compatibility metadata is generated from the driver itself
(`Describe()` — mandatory for every module) so it can't drift from the code:
the supported engine majors become the signed index's `engines` list, and the
SDK's `plugin.ProtocolVersion` becomes its `protocol`.

## How doze consumes it

Each module is published to its own rolling GitHub release (tag = module name)
serving a schema-1 module index + per-platform archives:

```
releases/download/<module>/index.yaml       # schema-1 (doze-sdk/modindex): per-release protocol,
                                            # supported engine majors, artifacts, channels
releases/download/<module>/meta.yaml        # docs prose, generated by `dzm meta` from Describe()
releases/download/<module>/<module>-plugin-<version>-<triple>.tar.gz   # contains bin/<module>-plugin
```

```yaml
# index.yaml
schema: 1
module: postgres
namespace: doze
releases:
  "0.2.0":
    protocol: 1                                # doze plugin protocol this binary speaks
    engines: ["14","15","16","17","18"]        # engine majors it supports (absent = versionless)
    artifacts:
      aarch64-apple-darwin: {url: …, sha256: …}
channels:
  stable: "0.2.0"
```

The registry's publish step signs each artifact's `sha256` AND the index itself
(so protocol/engine-support/channels can't be forged by the host). Publishing is
append-only per (version, triple): `dzm` refuses to re-publish a version with
different bytes — bump the version in `modules.yaml` instead.

doze resolves an engine type → module of the same name, selects a release from
the verified index, checks the archive's `sha256` + signature, and caches the
plugin. Override the source for development:

```sh
# a local dist dir (built by dzm) or any mirror base
export DOZE_MODULES_MIRROR=file:///path/to/dist
# or a single engine, straight to a binary, skipping fetch entirely
export DOZE_POSTGRES_PLUGIN=/path/to/postgres-plugin
```

## Building locally

```sh
go build -o /tmp/dzm ./cmd/dzm
/tmp/dzm --doze ../doze --out dist                     # all modules, all triples
/tmp/dzm --doze ../doze --out dist --module valkey     # one module
DOZE_MODULES_MIRROR=file://$PWD/dist doze up            # point doze at it
```
